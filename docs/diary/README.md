# 일별 예측 일기

매일 장마감 후 `데이터 수집 -> 예측 결과 검증 -> 모델 학습 -> 다음 거래일 예측 -> 오차 원인 기록` 순서로 작성합니다.

## 작성 원칙

| 항목 | 기록 내용 |
| --- | --- |
| 오늘 시장 결과 | 실제 강세/약세 섹터와 시장 국면 |
| 전일 예측 비교 | Top N 겹침, 방향 정확도, 행동 판단 |
| 데이터 수집 상태 | KIS, pykrx/KRX, NAVER, ECOS, OpenDART 상태 |
| 모델 학습 내용 | 오늘 결과로 모델이 새로 배운 점 |
| 다음 거래일 예측 | 핵심 관찰, 보조 관찰, 관망, 회피 우선 |

새 일기는 [TEMPLATE.md](TEMPLATE.md)를 복사해 `YYYY-MM-DD.md` 형식으로 작성합니다.

## 포트폴리오에서 읽는 법

이 일기는 단순 매매 기록이 아닙니다. 각 날짜는 모델 개선을 위한 실험 로그입니다.

- 맞은 날은 어떤 신호가 유효했는지 본다.
- 틀린 날은 어떤 변수나 게이트가 과하게 작동했는지 본다.
- 데이터 수집 실패는 모델 오류와 분리해서 기록한다.
- 매일 같은 형식으로 쌓아 장기적인 개선 과정을 보여준다.

전체 운영 규칙은 [일일 예측 일기 운영 가이드](../daily-prediction-diary.md)에서 관리합니다.
