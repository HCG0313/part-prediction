## 오늘 배운 점

오늘의 핵심은 모델이 틀렸다기보다, 실행 환경과 데이터 source 신뢰도를 더 엄격하게 관리해야 한다는 점입니다. 수집은 KIS 기준으로 정상화됐고, 학습도 로컬 Python 전환 후 성공했지만, KRX official 데이터는 여전히 partial 상태였습니다.

따라서 앞으로는 예측 결과만 기록하는 것이 아니라, 어떤 source가 실제로 활성화되었는지, 어떤 source는 fallback 또는 partial인지 함께 기록해야 합니다. 이 구조가 있어야 나중에 예측 실패가 모델 때문인지 데이터 신선도 때문인지 분리할 수 있습니다.
