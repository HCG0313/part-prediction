## 4. 오늘 해결한 문제

### KIS 연결 문제 원인 분리

테스트베드에서 토큰 발급과 기본시세 API 호출은 정상 처리되었다. 따라서 API 키 또는 토큰 발급 자체가 막힌 문제는 아니었다. 반면 로컬 PowerShell의 `Test-NetConnection openapi.koreainvestment.com -Port 9443`은 타임아웃을 보였고, 장중 수집 실패가 이 구간에서 발생했다.

결론적으로 오늘 문제는 `인증 실패`가 아니라 `장중 KIS endpoint 연결 불안정`으로 분리했다. 장마감 수집에서는 KIS가 정상 복구되었기 때문에, 향후에는 KIS가 불안정할 때 Naver realtime로 자동 전환하되 데이터 출처를 명확히 남기는 구조가 필요하다.

### KIS와 Naver 수집 파일 분리

KIS와 Naver 데이터를 같은 파일처럼 취급하면, 나중에 어떤 데이터로 예측했는지 추적하기 어렵다. 그래서 KIS 수집 결과와 Naver fallback 결과는 별도 파일로 저장하고, 실제 모델이 사용한 입력은 `active` 파일에 출처와 함께 기록하는 방향으로 정리했다.

이 구조의 목적은 단순 fallback이 아니라 재현성이다. 예측이 맞거나 틀렸을 때, 그날 모델이 KIS 가격을 봤는지, Naver 대체값을 봤는지 추적할 수 있어야 한다.

### 장중 데이터 품질 점검

일부 장중 데이터에서 시가, 고가, 저가가 0으로 들어오는 경우가 있어 수익률 계산에서 무한대나 이상값이 생길 수 있다. 오늘 개선 방향은 이런 행을 조용히 통과시키지 않고, 품질 이슈 파일에 원인과 조치 방향을 남기는 것이다.
