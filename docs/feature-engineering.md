# 특성공학과 전처리

## 목표

특성공학의 목표는 뉴스와 FOMO 기대효과를 그대로 믿는 것이 아니라, 실제 가격, 수급, 시장 국면과 결합해 "관심이 수익률로 이어질 가능성"을 판단하는 것입니다.

## 주요 특성 그룹

| 그룹 | 특성 예시 | 목적 |
| --- | --- | --- |
| FOMO/뉴스 | 뉴스 건수, 키워드 강도, 검색 관심도 | 시장 관심이 어디에 몰리는지 확인 |
| 가격 반응 | 섹터 수익률, 장중 수익률, open-to-now 수익률 | 관심이 실제 가격으로 연결됐는지 확인 |
| 시장 폭 | 상승 종목 비율, breadth delta | 일부 종목만 오른 것인지 섹터 전체가 오른 것인지 확인 |
| 수급/거래대금 | 거래대금 순위, 투자자 수급, 대형주 쏠림 | 실제 자금 유입 여부 확인 |
| 시장 국면 | risk_on, risk_off, capitulation, mixed_rotation | 같은 신호라도 장세에 따라 다르게 해석 |
| 캘린더 | 거래일, 휴장일, weekend gap | 잘못된 기준일과 중복 학습 방지 |

## 결측치 처리

결측치는 단순히 모두 0으로 채우지 않습니다. 결측의 의미가 변수마다 다르기 때문입니다.

| 결측 유형 | 처리 방향 |
| --- | --- |
| API 실패 | 상태값을 남기고 fallback 또는 제외 |
| 거래일 아님 | 가격 학습에서 제외 |
| 환율 기준일 지연 | 환율 상태값과 함께 보조 변수로 사용 |
| 장중 delta 계산 불가 | 결측으로 남긴 뒤 imputer가 처리 |
| 비정상 `inf` 값 | 결측으로 치환 후 학습 파이프라인에서 처리 |

## 이상치 처리

장중 데이터에서는 상한가, 하한가, 거래정지, 기준가 오류 등으로 이상치가 생길 수 있습니다. 현재 구조에서는 다음 기준을 우선합니다.

1. `inf`, `-inf`는 모델 입력 전에 결측으로 변환한다.
2. 거래일이 아닌 날짜의 가격 데이터는 학습에서 제외한다.
3. API 기준일이 수집일과 다르면 검증 기준 데이터로 바로 사용하지 않는다.
4. 거래대금 가중 수익률과 단순 평균 수익률을 함께 비교해 대형주 쏠림을 확인한다.

## 시장 국면 피처

같은 뉴스/FOMO 점수라도 장세에 따라 의미가 달라집니다.

| 국면 | 해석 |
| --- | --- |
| `risk_on` | 관심도와 가격 반응이 이어질 가능성이 높음 |
| `risk_off` | 관심도가 높아도 실제 수급이 약할 수 있음 |
| `capitulation` | 과매도 신호와 반등 신호를 구분해야 함 |
| `mixed_rotation` | 일부 섹터만 강하고 전체 확산은 제한적 |

## 독립변수와 종속변수

| 구분 | 내용 |
| --- | --- |
| 독립변수 | 전일 또는 장중까지 확인 가능한 뉴스, 검색, 가격, 수급, 환율, 공시, 국면 피처 |
| 종속변수 | 다음 거래일 섹터 수익률, 상승 여부, 상위 섹터 포함 여부 |
| 주의점 | 다음 날 결과를 당일 피처에 섞지 않도록 기준일을 엄격히 관리 |

## 최근 개선 내용

- 휴장일 또는 API 기준일 지연으로 같은 가격 데이터가 중복 학습되는 문제를 방지했습니다.
- ECOS 환율 데이터는 단순 값만 저장하지 않고 기준일과 수집 상태를 함께 남기도록 정리했습니다.
- 장중 피처에서 `inf`가 모델 입력으로 들어가는 문제를 방어 처리했습니다.
- KIS 실시간 수집 실패 시 Naver fallback을 사용하고 출처를 명확히 기록하는 방향으로 정리했습니다.
