## 2026-07-09: 후보 신뢰도 피드백과 베타 shadow 모델 비교

### 문제

2026-07-09 실제 장에서는 반도체/전자, 2차전지, 통신이 상위권이었습니다. 어제 예측 Top3는 반도체/전자, 조선/방산, 금융이었으므로 Top3 겹침은 1/3에 그쳤습니다.

특히 반도체/전자는 후보 보존이 맞았지만, 2차전지와 통신은 최종 행동층에서 낮게 잡혔고 금융은 방어 후보로 보았으나 실제로는 가장 약한 축에 가까웠습니다. 따라서 단순 점수 순위만 볼 것이 아니라 후보 신뢰도와 shadow 모델 성과를 같이 추적해야 했습니다.

### 해결

후보 신뢰도 성과 추적과 베타 모델 비교를 분리했습니다.

| 개선 항목 | 처리 내용 |
| --- | --- |
| 후보 신뢰도 성과 추적 | C/D 등급 후보가 실제 Top3에 들어갔는지 `candidate_confidence_performance` 계열 산출물로 기록 |
| 베타 라우터 shadow | 시장 국면별 가중치 조합을 별도 후보 모델로 추적 |
| 딥러닝 MLP shadow | 작은 신경망 모델을 베타 후보로 추가하되 메인 교체 없이 성과만 비교 |
| 모델 가중치 감사 | FOMO overlay와 shadow 조합 가중치를 매일 기록하고, 교체 조건을 통과할 때만 승격하도록 유지 |

### 검증

2026-07-09 장마감 파이프라인은 정상 완료되었습니다.

| 항목 | 결과 |
| --- | --- |
| KIS 마감 스냅샷 | 240개 종목 성공, 실패 0개, active source `kis_openapi` |
| KIS 수급 overlay | 238개 종목 성공, 2개 실패, `partial` |
| 평가 예측 수 | 420건 |
| 전체 방향 예측률 | 44.5% |
| 누적 Top3 겹침률 | 31.4% |
| 평균 RankIC | 0.031 |
| NDCG@3 | 0.586 |
| 수익률 구간 커버율 | 82.3% |
| 2026-07-09 당일 Top3 겹침 | 1/3 |

베타 shadow 비교 결과는 다음과 같습니다.

| 모델 | 검증 점수 | Top3 겹침률 | NDCG@3 | 판단 |
| --- | ---: | ---: | ---: | --- |
| 기존 메인 `baseline_main_total` | 0.495 | 36.2% | 0.587 | 현재 성능 리더 |
| `beta_router_shadow` | 0.487 | 33.3% | 0.584 | 성장 가능성 리더, 계속 관찰 |
| `beta_dl_mlp_shadow` | 0.420 | 18.8% | 0.543 | 표본 부족, 승격 보류 |

### 판단

현재는 메인 모델을 교체하지 않는 것이 맞습니다. 베타 라우터는 성장 가능성이 있지만 최근 5~10일 기준으로는 메인을 이기지 못했고, 딥러닝 MLP는 아직 표본 수와 안정성이 부족합니다. 따라서 메인 모델은 유지하고, 라우터와 딥러닝은 shadow 검증을 계속 누적합니다.

다음 개선 포인트는 2차전지와 통신처럼 실제 상위권이었지만 행동층에서 낮게 잡힌 섹터를 분석해, 후보 신뢰도 D 등급이 너무 강하게 후보를 누르는 상황을 줄이는 것입니다.

## 2026-07-06: 장마감 검증은 성공했지만 누적 품질 게이트 유지

### 문제

### 해결

## 2026-07-02: 시장 국면 라우터 추가

### 문제

기존 최종 점수는 시장이 상승장인지, 광범위 하락장인지, 급락 후 반등장인지에 따라 모델 가중치를 다르게 쓰지 못했습니다. 이 경우 하락장에서는 FOMO나 상승 확률이 과대 반영될 수 있고, 반대로 하락장 속 상대강도가 좋은 섹터를 충분히 설명하지 못할 수 있습니다.

### 해결

`build_tomorrow_prediction.py`에 `apply_regime_router_overlay()`를 추가했습니다. 이 함수는 기존 `tomorrow_total_score`를 덮어쓰지 않고, 시장 국면별 보조 점수인 `regime_router_score`를 계산해 작은 범위에서만 점수를 보정합니다.

| 국면 | 가중치 특징 |
| --- | --- |
| `risk_on_momentum` | FOMO와 모멘텀 비중 확대 |
| `broad_selloff` | 시장 압력, 리스크 방어, 상대강도 비중 확대 |
| `selloff_resilience` | 하락장 속 시장 대비 강한 섹터에 상대강도 가중 |
| `panic_rebound_watch` | 급락 후 반등 가능 구간에서 장중 반등과 상대강도 비중 확대 |
| `sector_rotation` | 섹터 순환장에서 FOMO와 상대강도 균형 반영 |
| `mixed_neutral` | 기본 균형 가중치 적용 |

### 검증

2026-07-02 15:35 실행 기준 라우터 분포입니다.

| 항목 | 결과 |
| --- | --- |
| `broad_selloff` | 7개 섹터 |
| `selloff_resilience` | 4개 섹터 |
| `panic_rebound_watch` | 1개 섹터 |

### 결론

시장 국면 라우터는 기존 메인 모델을 교체하는 것이 아니라, 현재 시장 환경에 맞게 어떤 신호를 더 믿을지 조정하는 보정층입니다. 오늘처럼 광범위 하락 압력이 강한 날에는 무조건 회피로만 몰지 않고, 시장 대비 상대강도가 살아 있는 섹터를 따로 분리해 설명할 수 있게 되었습니다.

## 2026-06-29: 메인/서브 모델 가중치 추적 문제

### 문제

| 기능 | 설명 |
| --- | --- |
| 가중치 상태 기록 | 현재 라이브 가중치와 섀도 추천 가중치를 `reports/model_weight_status.json`에 저장 |

| 섀도 예측표 | 추천 가중치로 `reports/model_weight_shadow_prediction.csv` 생성 |
| 해석 리포트 | `reports/model_weight_report.md`와 `reports/model_weight_shadow_prediction.md` 생성 |
