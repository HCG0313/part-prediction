# 일일 예측 일기 운영 가이드

이 문서는 날짜별 일기를 한 곳에 길게 중복 저장하는 문서가 아니라, 일일 예측 일기를 어떻게 작성하고 읽을지 안내하는 문서입니다.

실제 상세 기록은 [docs/diary/](diary/) 폴더의 날짜별 파일에 저장합니다.

## 운영 원칙

| 원칙 | 설명 |
| --- | --- |
| README는 포트폴리오 첫 화면 | 프로젝트 가치, 구조, 최신 성과, 핵심 문서 링크만 빠르게 보여준다 |
| 날짜별 일기는 `docs/diary/`에 저장 | 매일의 상세 분석은 `YYYY-MM-DD.md` 파일로 분리한다 |
| 같은 내용을 여러 문서에 반복하지 않음 | README에는 최근 핵심 요약만, 상세 근거는 날짜별 일기에서 관리한다 |
| 예측보다 검증을 중시 | 맞은 이유보다 틀린 이유와 모델이 배운 점을 더 자세히 기록한다 |
| 투자 추천처럼 쓰지 않음 | 모든 기록은 모델 검증과 개선을 위한 연구 일지로 작성한다 |

## 매일 작성 순서

1. 장마감 데이터 수집 상태를 확인한다.
2. 전일 예측과 오늘 실제 결과를 비교한다.
3. Top N 겹침, 방향 정확도, 행동 판단 실패 여부를 기록한다.
4. 데이터 수집 문제와 모델 판단 오류를 분리한다.
5. 모델이 새로 학습한 점을 정리한다.
6. 다음 거래일 예측을 작성한다.
7. README에는 최근 일기 요약만 필요할 때 갱신한다.

## 날짜별 일기 작성 형식

새 일기는 [일기 템플릿](diary/TEMPLATE.md)을 기준으로 작성합니다.

핵심 항목은 다음과 같습니다.

| 항목 | 기록 내용 |
| --- | --- |
| 오늘 시장 결과 | 실제 Top 섹터, 시장 국면, 섹터별 특징 |
| 전일 예측 비교 | 전일 예측 Top 섹터와 실제 Top 섹터의 차이 |
| 수집 상태 | KIS, pykrx/KRX, NAVER, ECOS, OpenDART, 글로벌 데이터 상태 |
| 오차 원인 | 데이터 문제인지, 모델 판단 문제인지 구분 |
| 모델 학습 | 오늘 결과로 조정해야 할 피처, 게이트, 페널티 |
| 다음 예측 | 핵심 관찰, 보조 관찰, 관망, 회피 우선 |

## 현재 일기 인덱스

날짜별 기록은 [일별 예측 일기 인덱스](diary/README.md)에서 확인합니다.

## 포트폴리오 관점에서의 의미

이 일기는 단순한 매매 기록이 아닙니다. 매일의 기록은 다음 질문에 답하기 위한 실험 로그입니다.

- 뉴스/FOMO 기대효과가 실제 섹터 수익률로 이어졌는가?
- 모델은 어떤 장세에서 잘 맞고 어떤 장세에서 틀렸는가?
- 데이터 수집 오류가 예측 성능을 왜곡했는가?
- 모델이 틀린 이유를 다음 개선으로 연결했는가?

따라서 좋은 일기는 "오늘 뭐가 올랐다"에서 끝나지 않고, "모델이 왜 그렇게 판단했고 무엇을 고쳐야 하는지"까지 남겨야 합니다.
