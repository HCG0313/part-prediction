# 문제 해결 기록

## 요약

이 프로젝트의 핵심 개선은 모델 알고리즘만 바꾼 것이 아니라, 데이터가 잘못 들어가는 상황을 찾아서 제어한 데 있습니다. 특히 휴장일, API 기준일 지연, 환율 수집 상태, 장중 비정상값 처리가 모델 안정성에 큰 영향을 줬습니다.

| 문제 | 증상 | 원인 | 해결 방향 |

## 휴장일 중복 학습 문제

가장 중요한 문제는 공휴일이나 휴장일에 API가 직전 거래일 데이터를 다시 반환하는 상황이었습니다. 이 데이터가 그대로 들어가면 모델은 같은 가격 흐름을 다른 날짜 데이터로 착각하고 중복 학습할 수 있습니다.

해결 방향은 다음과 같습니다.

1. 오늘이 실제 거래일인지 먼저 확인한다.
2. API가 반환한 기준일이 수집 기준일과 일치하는지 확인한다.
3. 장이 열리지 않은 날은 가격 학습을 중단한다.
4. 뉴스/FOMO 데이터는 다음 실제 거래일 예측 후보로만 남긴다.

## 환율 데이터 문제

환율은 주식시장과 완전히 같은 휴장일 규칙을 쓰지 않습니다. 그래서 환율은 "수집 성공/실패"보다 "어떤 기준일의 환율인지"가 더 중요합니다.

현재 정리 방향은 다음과 같습니다.

| 항목 | 처리 |
| --- | --- |
| 환율 값 | 모델 보조 변수로 사용 |
| 환율 기준일 | 반드시 함께 저장 |
| 캐시 사용 여부 | 상태값으로 기록 |
| 수집 실패 | 이전 값 사용 여부를 명시 |

## 장중 반등 모델 문제

장중 반등 모델은 정확도만 보면 높아 보일 수 있습니다. 하지만 반등 이벤트 자체가 적으면 대부분을 "반등 아님"으로 예측해도 정확도가 높아지는 착시가 생깁니다.

따라서 이 모델은 accuracy보다 다음 지표를 더 중요하게 봐야 합니다.

- precision
- recall
- roc_auc
- 실제 강한 반등 후보를 놓친 사례

2026-06-12 기준 장중 반등 모델은 accuracy는 높지만 precision/recall이 약해, 아직 후보 포착 모델로는 개선이 필요합니다.

## 남은 개선 과제

1. KRX 공식 데이터 지연이 반복될 때 검증 데이터 우선순위를 자동 결정한다.
2. 장중 `inf` 방어 처리를 모든 파생 변수 생성 단계에 확장한다.
3. 뉴스/FOMO가 과열 신호인지 반등 전조인지 구분하는 별도 피처를 만든다.
4. 강한 반등장 초입에서 회피 게이트가 과하게 작동하는 문제를 줄인다.
